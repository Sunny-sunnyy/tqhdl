1. Tổng quan Dự án
1.1. Mô tả dự án
USA Regional Sales Analysis là một dự án portfolio toàn diện về phân tích dữ liệu bán hàng theo vùng miền tại Mỹ (2014–2018). Dự án bao trùm toàn bộ quy trình Data Analytics: từ thu thập dữ liệu, làm sạch, chuyển đổi, phân tích khám phá (EDA) bằng Python, đến xây dựng dashboard trên Power BI. Mục tiêu là xác định các yếu tố thúc đẩy doanh thu và lợi nhuận, phát hiện xu hướng theo mùa, và hỗ trợ ra quyết định chiến lược.
1.2. Problem Statement
Các đội sales thường thiếu cái nhìn dựa trên dữ liệu về hiệu suất từng vùng, khiến cho việc xác định cơ hội tăng trưởng và tối ưu tài nguyên trở nên khó khăn. Dự án phân tích dữ liệu sales 2014–2018 để xác định các yếu tố chính thúc đẩy doanh thu và lợi nhuận qua sản phẩm, kênh bán hàng, và vùng miền; phát hiện xu hướng theo mùa và outliers; đối chiếu hiệu suất với ngân sách. Từ đó tối ưu giá, khuyến mãi và mở rộng thị trường.
1.3. Mục tiêu cuối cùng
•	Xác định sản phẩm, kênh bán hàng, và vùng miền có hiệu suất cao nhất
•	Phát hiện xu hướng theo mùa và các bất thường trong dữ liệu
•	Phân khúc khách hàng theo doanh thu và lợi nhuận
•	Xây dựng dashboard Power BI tương tác để stakeholders sử dụng hàng ngày
1.4. Đầu vào và Đầu ra
Đầu vào (Input): File Excel (Regional_Sales_Dataset.xlsx) chứa 6 sheet dữ liệu thô: Sales Orders (64,104 records), Customers (175 records), Products (30 records), Regions (994 records), State Regions (49 records), 2017 Budgets (30 records).
Đầu ra (Output): (1) File CSV đã làm sạch và tạo features mới (Sales_data_EDA_Exported_.csv – 64,104 records, 21 cột). (2) Notebook EDA với 15 phân tích và insights. (3) Dashboard Power BI 3 trang (SALES_REPORT.pbix). (4) Slide tóm tắt dự án (PPT).

3. Workflow End-to-End của Dự án
3.1. Giai đoạn 1: Business Understanding
Hiểu bài toán kinh doanh: phân tích dữ liệu sales theo vùng miền để tìm cơ hội tăng trưởng, tối ưu nguồn lực, và hỗ trợ ra quyết định. Tưởng tượng bạn làm cho một công ty e-commerce và đây là dữ liệu sales của họ.
3.2. Giai đoạn 2: Data Understanding & ERD
Phân tích Entity Relationship Diagram để hiểu cấu trúc dữ liệu. Xác định Primary Key và Foreign Key của từng bảng:
•	Sales Orders: PK = Order Number; FK = Customer Name Index, Delivery Region Index, Product Description Index
•	Customers: PK = Customer Index → liên kết với Sales Orders
•	Products: PK = Index → liên kết với Sales Orders; Product Name → liên kết với 2017 Budgets
•	Regions: PK = id → liên kết với Sales Orders qua Delivery Region Index
•	State Regions: PK = State Code → liên kết với Sales Orders
•	2017 Budgets: liên kết với Products qua Product Name
Quan hệ: Customer → Order (1:N), Region → Order (1:N), Product → Order (1:N). Sales Orders là fact table trung tâm, các bảng khác là dimension tables.
3.3. Giai đoạn 3: Data Collection
Dữ liệu từ file Excel công khai (Kaggle). Trong thực tế có thể đến từ database (Snowflake, MongoDB, PostgreSQL), data warehouse, hoặc API. Sử dụng pd.read_excel(file_path, sheet_name=None) để đọc tất cả 6 sheet cùng lúc.
3.4. Giai đoạn 4: Data Cleaning & Wrangling
Bước này thực hiện trong notebook (Cells 15–24), bao gồm:
1.	Sửa header bị lệch ở bảng State Regions (dòng đầu tiên chứa tên cột thật)
2.	Kiểm tra null values và duplicates
3.	Merge 6 bảng thành 1 DataFrame duy nhất bằng left join (5 lần merge liên tiếp)
4.	Loại bỏ cột thừa sau merge (Customer Index, Index, id, State Code)
5.	Chọn cột cần giữ dựa trên domain knowledge (15 cột)
6.	Đổi tên cột thành snake_case cho nhất quán
7.	Xử lý budget: gán null cho các năm không phải 2017
3.5. Giai đoạn 5: Feature Engineering
Tạo các cột mới từ dữ liệu có sẵn (Cell 26):
•	total_cost = quantity × cost (tổng chi phí mỗi dòng)
•	profit = revenue – total_cost (lợi nhuận)
•	profit_margin_pct = (profit / revenue) × 100 (tỷ lệ lợi nhuận %)
•	order_month_name = tên tháng (January, February...) – dùng cho biểu đồ
•	order_month_num = số tháng (1–12) – dùng để sort đúng thứ tự trong Power BI

Trang 2 – Product & Channel: Bar charts (Product Revenue, Product Margin) + Channel analysis + Text insights

DAX Measures đã tạo (trong bảng Custom Measures):
Profit Margin % = DIVIDE(SUM(profit), SUM(revenue), 0) * 100
Revenue Per Order = DIVIDE(SUM(revenue), COUNT(order_number))
Tính năng đặc biệt: Page Navigation (buttons chuyển trang), Filter Panel (ẩn/hiện bằng Ctrl+Click), Bookmarks (Top 5 / Bottom 5 toggle), Clear All Slicers button.

6.1. Import & Load (Cells 8–10)
# Import thư viện xử lý dữ liệu và trực quan hóa
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

# Đọc tất cả sheet từ Excel (sheet_name=None → trả về dict)
sheets = pd.read_excel(file_path, sheet_name=None)

# Gán từng sheet vào DataFrame riêng
df_sales     = sheets['Sales Orders']    # 64,104 records
df_customers = sheets['Customers']        # 175 records

Mục đích: Tải toàn bộ dữ liệu từ file Excel nhiều sheet vào các DataFrame độc lập để xử lý.
6.2. Merge Pipeline (Cell 19)
# Merge 5 lần liên tiếp để tạo DataFrame hoàn chỉnh
df = df_sales.merge(df_customers, how='left',
    left_on='Customer Name Index',
    right_on='Customer Index')
df = df.merge(df_products, how='left',
    left_on='Product Description Index', right_on='Index')
df = df.merge(df_regions, how='left',
    left_on='Delivery Region Index', right_on='id')
df = df.merge(df_state_reg[['State Code','Region']],
    how='left', left_on='state_code', right_on='State Code')

Mục đích: Tạo một DataFrame duy nhất chứa đầy đủ thông tin từ 6 bảng. Luôn dùng left join để giữ toàn bộ records từ fact table (Sales Orders).
6.3. Feature Engineering (Cell 26)
# Tạo cột tổng chi phí, lợi nhuận, tỷ lệ lợi nhuận
df['total_cost'] = df['quantity'] * df['cost']
df['profit'] = df['revenue'] - df['total_cost']
df['profit_margin_pct'] = (df['profit']/df['revenue'])*100

# Trích xuất thông tin tháng cho biểu đồ và Power BI
df['order_month_name'] = df['order_date'].dt.month_name()
df['order_month_num'] = df['order_date'].dt.month
Mục đích: Biến dữ liệu thô thành thông tin phân tích được. Cặp order_month_name + order_month_num rất quan trọng cho Power BI (dùng Sort by Column).
6.4. Xử lý dữ liệu 2018 (Cell 32)
# Loại bỏ 2018 (chỉ có tháng 1-2, làm lệch phân tích)
df_ = df[df['order_date'].dt.year != 2018]
Mục đích: Dữ liệu 2018 không đầy đủ (chỉ 2 tháng) sẽ làm sai lệch phân tích temporal khi gộp tất cả các năm. Đây là bài học quan trọng về incomplete data.
6.5. Export CSV (Cell 75)
# Xuất DataFrame đã xử lý để dùng trong Power BI
df.to_csv('Sales_data(EDA Exported).csv', index=False)
Mục đích: Tạo cầu nối giữa Python (EDA) và Power BI (dashboard). File CSV này chính là data source cho toàn bộ dashboard.
